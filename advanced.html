Suggested learning path

Three tiers. Tier 1 builds on agentic-inbox directly; Tier 2 is product depth; Tier 3 is production hardening.

Tier 1 · Weeks 1–2

Agent stack (study agentic-inbox as-is)
  1. Deploy agentic-inbox with Workers AI enabled
  2. Trace the 9 email tools in workers/lib/tools.ts
  3. Use the agent sidebar: summarize, search, draft — observe confirm-before-send
  4. Connect Cursor to /mcp; list mailboxes and read mail via MCP
  5. Read Email for Agents blog post end to end

Checkpoint: Agent drafted a reply you confirmed and sent; MCP tool call succeeded from Cursor.

Tier 2 · Weeks 3–5

Product features on your clone
  1. Drafts table + autosave debounce in compose UI
  2. Custom labels (many-to-many message ↔ label)
  3. Inbound filter rules evaluated in consumer Worker
  4. SSE or Web Push for new-mail notifications
  5. Alias CRUD: multiple addresses routing to one mailbox DO

Checkpoint: Filter auto-archives a newsletter; draft survives page refresh; push fires on inbound test.

Tier 3 · Weeks 6+

Infrastructure & standards
  1. Refactor inbound to Queue → consumer parse (GUIDE.md pattern)
  2. Bounce/complaint webhooks + suppression list + dashboard
  3. Weekly backup Worker: export D1/R2 to cold bucket
  4. Read RFC 8621 (JMAP); document gap analysis vs your schema
  5. Spike: read-only IMAP SELECT/FETCH against your message store
  6. Multi-user auth model (replace single Access policy)

Checkpoint: Parse retry survives intentional throw; mbox export opens in Thunderbird.
